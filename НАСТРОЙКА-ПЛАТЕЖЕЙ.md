# 🚀 Настройка системы оплаты для продакшена

## 1. ЮKassa (Яндекс.Касса) - для карт

### Регистрация:
1. Перейти на https://yookassa.ru/
2. Зарегистрировать магазин
3. Пройти модерацию (обычно 1-3 дня)
4. Получить данные в личном кабинете

### Настройки в .env:
```env
YOOKASSA_SHOP_ID=123456  # ID магазина из личного кабинета
YOOKASSA_SECRET_KEY=live_ABC123...  # Секретный ключ из личного кабинета
```

### Webhook URL для уведомлений:
```
https://ваш-домен.ru/api/payments/webhook/yookassa
```

### Комиссия: 
- 2.8% + 15₽ за успешный платеж картой

---

## 2. СБП (Система быстрых платежей) - для QR-кодов

### Подключение через банк:
1. **Сбербанк**: https://www.sberbank.ru/ru/s_m_business/bankingservice/acquiring_sbp
2. **Тинькофф**: https://www.tinkoff.ru/business/acquiring/sbp/
3. **Альфа-Банк**: https://alfabank.ru/sme/acquiring/sbp/

### Настройки в .env:
```env
SBP_MERCHANT_ID=ваш_merchant_id  # ID мерчанта от банка
SBP_SECRET_KEY=ваш_secret_key    # Секретный ключ от банка
```

### Комиссия:
- 0.4-0.7% за платеж через СБП

---

## 3. Настройка домена и SSL

### В .env:
```env
APP_URL=https://ваш-домен.ru  # Реальный домен с SSL
```

### Требования:
- ✅ HTTPS обязательно для платежей
- ✅ Валидный SSL сертификат
- ✅ Домен должен быть доступен извне

---

## 4. Webhook endpoints

Убедитесь что эти URL доступны:

```
POST https://ваш-домен.ru/api/payments/webhook/yookassa
```

### Настройка в ЮKassa:
1. Личный кабинет → Настройки → HTTP-уведомления
2. URL: `https://ваш-домен.ru/api/payments/webhook/yookassa`
3. События: `payment.succeeded`, `payment.canceled`

---

## 5. Тестирование

### Тестовые карты ЮKassa:
- **Успешная оплата**: `5555555555554444`
- **Отклоненная оплата**: `5555555555554477`
- CVV: любой, срок: любой будущий

### Проверка:
```bash
# Проверить доступность webhook
curl -X POST https://ваш-домен.ru/api/payments/webhook/yookassa

# Проверить создание платежа
curl -X POST https://ваш-домен.ru/api/payments/orders/test/pay \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ваш-jwt-token" \
  -d '{"payment_method": "card"}'
```

## 6. Мониторинг

### Логи платежей:
- Все платежи логируются в консоль
- Ошибки сохраняются с префиксом `[payment]`

### Важные метрики:
- Конверсия платежей
- Время обработки webhook
- Количество отклоненных платежей

## 💡 Дополнительные возможности

### Возвраты:
Система поддерживает возвраты через API:
```javascript
await paymentService.refundPayment(paymentId, amount, reason);
```

### Рассрочка:
ЮKassa поддерживает рассрочку - можно добавить как отдельный метод оплаты.

### Сохраненные карты:
Можно добавить сохранение карт для постоянных клиентов.
