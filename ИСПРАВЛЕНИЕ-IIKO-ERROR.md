# Исправление ошибки iiko API

## Проблема
```
iiko menu/by_id error (401): Right api/1/menu/by_id is not allowed for this ApiLogin
```

Это означает, что ваш API-логин не имеет прав на доступ к внешнему меню.

## Решение 1: Использовать номенклатуру (РЕКОМЕНДУЕТСЯ)

### Шаг 1: Удалите переменную IIKO_EXTERNAL_MENU_ID из Railway

1. Откройте Railway: https://railway.app
2. Выберите проект "satory-backend"
3. Перейдите во вкладку "Variables"
4. Найдите переменную `IIKO_EXTERNAL_MENU_ID`
5. Нажмите на неё и выберите "Remove"
6. Railway автоматически перезапустит сервис

### Шаг 2: Запустите синхронизацию снова

```bash
node sync-railway-iiko.js
```

Или через curl:
```bash
curl -X POST https://satory-backend-production.up.railway.app/api/iiko/sync/api \
  -H "x-admin-secret: satory_admin_2026"
```

✅ Теперь система будет использовать обычную номенклатуру iiko

---

## Решение 2: Дать права API-логину на внешнее меню

Если вам нужно именно внешнее меню:

### Шаг 1: Зайдите в личный кабинет iiko
1. Откройте https://iiko.net
2. Войдите в личный кабинет

### Шаг 2: Настройте права API-логина
1. Перейдите в раздел "API" или "Интеграции"
2. Найдите ваш API-логин
3. Убедитесь, что у него есть права:
   - ✅ Чтение номенклатуры
   - ✅ Доступ к внешнему меню (menu/by_id)

### Шаг 3: Проверьте IIKO_EXTERNAL_MENU_ID
Убедитесь, что в Railway переменная `IIKO_EXTERNAL_MENU_ID` содержит правильный ID внешнего меню.

### Шаг 4: Запустите синхронизацию
```bash
node sync-railway-iiko.js
```

---

## Что делает каждый метод

### Номенклатура (по умолчанию)
- Использует эндпоинт `/api/1/nomenclature`
- Получает все товары и категории из iiko
- Работает с базовыми правами API

### Внешнее меню
- Использует эндпоинт `/api/1/menu/by_id`
- Получает товары из настроенного внешнего меню
- Требует дополнительных прав API
- Полезно, если у вас несколько меню (например, для разных точек)
