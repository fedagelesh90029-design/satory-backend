# Настройка iiko API в Railway

## Шаг 1: Получите данные от iiko

1. Зайдите на https://iiko.net
2. Войдите в личный кабинет iiko Cloud
3. Перейдите в раздел **API** или **Интеграции**
4. Создайте новый API-логин (если еще не создан)
5. Скопируйте:
   - **API Login** (например: `api_login_12345`)
   - **Organization ID** (UUID, например: `a1b2c3d4-e5f6-7890-abcd-ef1234567890`)

## Шаг 2: Добавьте переменные в Railway

1. Откройте ваш проект в Railway: https://railway.app
2. Выберите сервис **backend**
3. Перейдите во вкладку **Variables**
4. Добавьте следующие переменные:

### Обязательные переменные:

```
IIKO_API_LOGIN=ваш_api_логин_от_iiko
IIKO_ORGANIZATION_ID=ваш_uuid_организации
```

### Опциональные переменные (уже есть значения по умолчанию):

```
IIKO_API_URL=https://api.iiko.ru/api/1
IIKO_SYNC_CRON=0 0,6,12,18 * * *
```

**Примечание:** `IIKO_SYNC_CRON` - расписание автоматической синхронизации:
- `0 0,6,12,18 * * *` = каждые 6 часов (в 00:00, 06:00, 12:00, 18:00)
- Можете изменить на свое расписание

## Шаг 3: Перезапустите сервис

После добавления переменных Railway автоматически перезапустит сервис.

## Шаг 4: Проверьте подключение

### Через API:

```bash
# Проверить статус
curl https://ваш-домен.railway.app/api/iiko/status

# Запустить синхронизацию вручную
curl -X POST https://ваш-домен.railway.app/api/iiko/sync/api \
  -H "x-admin-secret: satory_admin_2026"
```

### Через админ-панель:

1. Откройте админ-панель: `https://ваш-домен.railway.app/admin.html`
2. Войдите с логином и паролем
3. Перейдите в раздел **Синхронизация**
4. Нажмите кнопку **Синхронизировать с iiko**

## Как работает синхронизация

### Автоматическая синхронизация:
- Запускается по расписанию (по умолчанию каждые 6 часов)
- Обновляет товары и категории из iiko
- Не затрагивает ручные переопределения цен и категорий

### Ручная синхронизация:
```bash
curl -X POST https://ваш-домен.railway.app/api/iiko/sync/api \
  -H "x-admin-secret: satory_admin_2026"
```

### Что синхронизируется:
- ✅ Товары (название, цена, описание)
- ✅ Категории
- ✅ Изображения (если есть в iiko)
- ✅ Статус активности товаров
- ❌ Не затрагивает `price_override` и `category_override` (ручные переопределения)

## Логи синхронизации

Проверить логи синхронизации:
```bash
curl https://ваш-домен.railway.app/api/admin/sync/status \
  -H "x-admin-secret: satory_admin_2026"
```

## Troubleshooting

### Ошибка "iiko auth failed (404)"
- Проверьте правильность `IIKO_API_LOGIN`
- Убедитесь, что API-логин активен в личном кабинете iiko

### Ошибка "IIKO_API_LOGIN не задан"
- Убедитесь, что переменная добавлена в Railway
- Проверьте, что сервис перезапущен после добавления переменной

### Товары не синхронизируются
- Проверьте `IIKO_ORGANIZATION_ID` - должен быть UUID вашей организации
- Убедитесь, что в iiko есть активные товары
- Проверьте логи в Railway

## Дополнительно: Внешнее меню

Если у вас настроено внешнее меню в iiko, добавьте:
```
IIKO_EXTERNAL_MENU_ID=ваш_external_menu_id
```

Тогда синхронизация будет использовать внешнее меню вместо номенклатуры.
